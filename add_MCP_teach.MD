# Model Context Protocol (MCP) คืออะไร?

เอกสารนี้จะอธิบายแนวคิดของ Model Context Protocol (MCP) แบบง่ายๆ สำหรับผู้เริ่มต้น

## MCP คืออะไร?

**Model Context Protocol (MCP)** คือ **"มาตรฐานกลาง"** หรือ **"ภาษากลาง"** ที่ออกแบบมาเพื่อให้โปรแกรม AI (หรือที่เรียกว่า Large Language Models - LLMs) สามารถสื่อสารและทำงานร่วมกับเครื่องมือ (Tools) หรือแหล่งข้อมูลภายนอกได้

ลองนึกภาพว่า AI ก็เหมือนกับ "สมอง" ที่มีความสามารถในการคิดวิเคราะห์และสนทนา แต่ถ้ามันต้องการข้อมูลจากโลกภายนอก หรือต้องการสั่งให้โปรแกรมอื่นทำงาน มันจะทำอย่างไร? MCP คือสะพานที่เชื่อมระหว่าง "สมอง AI" กับ "เครื่องมือ" เหล่านั้นนั่นเอง

## ทำไม MCP ถึงมีประโยชน์?

โดยปกติแล้ว AI จะถูกจำกัดอยู่แค่ในโลกของข้อมูลที่มันถูกฝึกมา แต่ MCP จะช่วยทลายกำแพงนั้นออกไป ทำให้ AI:

1.  **เข้าถึงข้อมูลสดใหม่ (Real-time Data)**: AI สามารถดึงข้อมูลล่าสุดจากฐานข้อมูล, API, หรือแม้กระทั่งไฟล์ในเครื่องคอมพิวเตอร์ของเราได้
2.  **สั่งการโปรแกรมอื่นได้ (Take Actions)**: AI สามารถสั่งให้โปรแกรมอื่นทำงานได้ เช่น สั่งให้สแกนไฟล์, ค้นหาข้อมูลในระบบ, หรือส่งอีเมล
3.  **ทำงานร่วมกับเครื่องมือที่หลากหลาย**: ไม่ว่าเครื่องมือจะเขียนด้วยภาษาอะไร (Go, Python, JavaScript) ถ้ามันรองรับ "ภาษากลาง" ของ MCP ตัว AI ก็จะสามารถคุยกับมันได้ทั้งหมด

**เปรียบเทียบง่ายๆ:**
MCP ก็เหมือนกับ **"รีโมทคอนโทรลอัจฉริยะ"**
-   **AI**: คือคนที่ถือรีโมท
-   **เครื่องมือ (Tools)**: คืออุปกรณ์ต่างๆ เช่น ทีวี, แอร์, หรือเครื่องเสียง
-   **MCP**: คือสัญญาณอินฟราเรด (Infrared) ที่เป็นมาตรฐาน ทำให้รีโมทอันเดียวสามารถควบคุมอุปกรณ์ได้ทุกยี่ห้อ

ในโปรเจกต์ของเรา การเพิ่ม MCP เข้าไป ก็เหมือนกับการสร้าง "รีโมท" ให้ AI สามารถสั่ง "ค้นหาไฟล์" หรือ "ดูข้อมูลกราฟ" ในโปรแกรม File Graph Visualizer ของเราได้โดยตรง

## มันทำงานอย่างไร?

การทำงานของ MCP มีองค์ประกอบหลักๆ คือ:

1.  **MCP Server (ฝั่งเครื่องมือ)**:
    -   นี่คือโปรแกรมที่เราจะสร้างขึ้น (ในที่นี้คือ File Graph Visualizer ของเรา)
    -   มันจะ "ประกาศ" ว่า "ฉันมีเครื่องมือ (Tools) เหล่านี้ให้ใช้" เช่น `getNodeCount`, `searchFile`
    -   มันจะรอรับคำสั่งจาก AI

2.  **MCP Client (ฝั่ง AI)**:
    -   นี่คือตัว AI หรือ LLM
    -   เมื่อ AI ต้องการใช้เครื่องมือ มันจะส่ง "คำขอ" (Request) ไปยัง MCP Server
    -   ตัวอย่างเช่น: "ขอใช้เครื่องมือ `searchFile` ค้นหาคำว่า `report.pdf`"

3.  **Transport (ช่องทางการสื่อสาร)**:
    -   คือช่องทางที่ Client กับ Server ใช้คุยกัน
    -   ช่องทางที่นิยมคือ **Stdio (Standard Input/Output)** ซึ่งเป็นการสื่อสารผ่าน Text Command Line เหมาะสำหรับโปรแกรมที่รันบนเครื่องคอมพิวเตอร์เดียวกัน

ดังนั้น เมื่อเราเพิ่ม MCP เข้าไปในโปรแกรม เรากำลังสร้าง **MCP Server** ที่รอรับคำสั่งจาก AI เพื่อให้ AI สามารถเข้ามาควบคุมและใช้ความสามารถของโปรแกรมเราได้นั่นเอง
